## LightND - Release v8.5.1

### Improvements:

### Instructions:
高级搜索支持的类不等式变量值及说明如下表：

|     值     |          说明           |
|:---------:|:---------------------:|
| `writer`  |    筛选作者，仅支持`=`运算符     |
|  `bunko`  |    筛选文库，仅支持`=`运算符     |
|   `tag`   |    筛选标签，仅支持`=`运算符     |
| `numanme` |        筛选数字编码         |
|  `liked`  | 筛选喜欢状态，仅支持`0`和`1`作为参数 |
|  `star`   |         筛选评分          |
现在支持的运算符仅有`=`, `>`, `<`。这意味着你暂时不能使用`>=`和`<=`，这将在未来被解决。
在使用高级搜索时需在字符串头加入` `以声明。
任何不符合语法的类不等式将被视作关键字。

用例：
 - ` star>5.12 tag=JK wode` -> 这意味着你在搜索库中评分大于`5.12`，有`JK`标签的，关键字为`wode`的书。
 - ` numname>4000 numname<3500 star=` ->  这意味着你在搜索库中数字编码大于`3500`且小于`4000`的，关键字为`star=`的书。

## LightND - Release v8.2

### What's New:
 - 代理端口指定。你现在可以使用新增的配置项`PROXY_PORT`来设置发送`GET`请求时使用的代理端口。该配置项接受三种值，说明见下方。

### Improvements:
 - 现在`CONFIG`中的整型输入有一个`-65536`-`65536`的范围限制。

### Bug Fixes:
 - 修复了特定情况下配置项设置失效的问题。
 - 修复了其他已知问题。

### Instructions:
`PROXY_PORT`配置项支持的值及说明如下表：

|   值   |         说明         |
|:-----:|:------------------:|
| `-1`  |      不设置代码层代理      |
|  `0`  |   仅对非插图下载设置一空代理    |
| 其余正值  | 对所有请求设置端口为该值的代码层代理 |

以`Clash For Windows`(`CFW`)为例，当设置该配置项值为`-1`时，所有请求发送按默认方式，即是否流经`CFW`取决于是否打开其系统代理；当设置该配置项值为`0`时，所有非插图请求全部不流经`CFW`，无论系统代理是否开启；当设置该配置项值为`7890`(即`CFW`的默认监听端口)时，所有请求均流经`CFW`，无论其系统代理是否打开。
 - 值得注意的是，“流经`CFW`”并不意味着这些请求或流量一定从`CFW`的监听端口发出，若更改配置项设置无效，你可以尝试查看并更改`CFW`的代理规则。
