## LightND - Release v8.3

### What's New:
 - 多线程启动。现在书库启动时会使用多线程的数据读取方式，这将大幅加速启动速度。这一点在CPU频率较低的设备上较明显，加速最高达到约50%。
 - 细节菜单(`ctask`)优化。现在细节菜单的`Label`和其对应复选框总是有同样的功能(这以前只在`ChapterWidget`上被使用)。特别地，对可以展开子菜单的组件，其左键点击对应展开和收起(即左侧箭头)，右键点击对应复选框操作。

### Improvements:
 - 格式化的反馈信息现在更加清晰。在终端中现在只会显示一本书的格式化参数，且书名可见。
 - 细节菜单的UI翻新。

### Bug Fixes:
 - 修复了部分情况下全选按钮的刷新错误的问题。
 - 修复了特定情况下三态复选框行为不符合预期的问题。


## LightND - Release v8.2.1

## LightND - Release v8.2

### What's New:
 - 代理端口指定。你现在可以使用新增的配置项`PROXY_PORT`来设置发送`GET`请求时使用的代理端口。该配置项接受三种值，说明见下方。

### Improvements:
 - 现在`CONFIG`中的整型输入有一个`-65536`-`65536`的范围限制。

### Bug Fixes:
 - 修复了特定情况下配置项设置失效的问题。
 - 修复了其他已知问题。

### Instructions:
`PROXY_PORT`配置项支持的值及说明如下表：

|   值   |         说明         |
|:-----:|:------------------:|
| `-1`  |      不设置代码层代理      |
|  `0`  |   仅对非插图下载设置一空代理    |
| 其余正值  | 对所有请求设置端口为该值的代码层代理 |

以`Clash For Windows`(`CFW`)为例，当设置该配置项值为`-1`时，所有请求发送按默认方式，即是否流经`CFW`取决于是否打开其系统代理；当设置该配置项值为`0`时，所有非插图请求全部不流经`CFW`，无论系统代理是否开启；当设置该配置项值为`7890`(即`CFW`的默认监听端口)时，所有请求均流经`CFW`，无论其系统代理是否打开。
 - 值得注意的是，“流经`CFW`”并不意味着这些请求或流量一定从`CFW`的监听端口发出，若更改配置项设置无效，你可以尝试查看并更改`CFW`的代理规则。


## LightND - Release v8.1

### What's New:
 - 后缀模板。你现在可以使用新增的配置项`RMZ_FILENAME_SUFFIX`来调整后缀的格式。该字符串将被以第一个`%NUM%`分割，将前份作为后缀前份，后份作为后缀后份。
 - 文本格式化组件支持部分切片功能。现在可以在已获支持的任意文本后加上`[n]`，其中`n`为任意整数。正整数将使其被视作`slice(0, n, None)`，负整数将使其被视作`slice(n, None, None)`。

### Bug Fixes:
 - 修复了特定情况下文本格式化组件不起效的问题。
 - 修复了其他已知问题。

### Instructions:
当前已获支持的文本格式化组件如下表：

|     名称     |    说明    |
|:----------:|:--------:|
| `TMSTAMP`  |  当前的时间戳  |
| `NUMNAME`  |  书的数字编码  |
|  `BKNAME`  |    书名    |
| `DATETIME` | 当前的时间字符串 |




## LightND - Release v8.0

### What's New:
 - 书籍结构化重整。现在书籍相关信息将更好地分类存储，这将以略微增加`bank.json`大小为代价，显著降低`HFolder`中`.hmz`文件的大小，并略微加速各处运算。
 - 书库文件存储重做。现在你可以通过调整新增的配置项`SIMPLE_BANK_FILE`来调整`bank.json`的存储方式，使之更易读或节省空间。将该选项调整为`False`可节省40%-60%的空间。
 - 新的书库排序项。你现在可以按入库时间来为书库排序，更新前入库的书籍将按照其`HFolder`中`hmz`文件的创建时间计算。

### Improvement:
 - 书籍组件优化。现在书籍组件不再依赖`bank.json`更新，这将显著加速书籍信息的更新，并加快书库页面的生成与刷新。
 - 书库筛选项优化。现在书库的筛选更加有序，这将小幅提升筛选速度。
 - 下载流程优化。现在下载流程被重新排序，这对性能没有可预见的影响，但减少了可能发生的潜在错误。
 - `rmz`文件导出优化。
 - `%TIMESTAMP%`文本格式化组件现在仅记录到秒精度。
 - 其他代码重构及优化。

### Bug Fixes:
 - 现在`config.json`不再记录仓库中。
 - `rmz`文件现在不再因导出路径缺失或文件名已存在而导出失败。若发生后者，其文件名后将出现后缀。该后缀将在未来计划中变为可修改的配置项。
 - 修复了某些书籍信息在特定情况下丢失的问题。
 - 修复了其他已知问题。
